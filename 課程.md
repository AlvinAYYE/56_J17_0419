## Vue 基本啟動觀念

最基本的結構長這樣：

```html
<script src="./vue.3.5.13.js"></script>

<div id="app">
  <h1>{{ msg }}</h1>
</div>

<script>
const { createApp, ref } = Vue

createApp({
  setup() {
    const msg = ref('你好，Vue')
    return { msg }
  }
}).mount('#app')
</script>
```

### 這段在做什麼

1. 先引入 Vue
2. 找到 `#app`
3. 在 `setup()` 裡建立資料
4. 用 `return` 把資料交給畫面
5. 畫面用 `{{ msg }}` 顯示

### 組合式 API 的閱讀順序

看組合式 API 時，建議固定用這個順序：

1. 先看 `setup()`
2. 找 `ref()` 和陣列資料
3. 找 function
4. 看 `return` 有交出去什麼
5. 回頭看 HTML 用到哪些名字

## 組合式 API 與選項式 API 的快速差別

### 選項式 API 常見外觀

```js
createApp({
  data() {
    return {
      msg: '早安'
    }
  },
  methods: {
    changeMsg() {
      this.msg = '下午好'
    }
  }
}).mount('#app')
```

### 組合式 API 常見外觀

```js
createApp({
  setup() {
    const msg = ref('早安')

    function changeMsg() {
      msg.value = '下午好'
    }

    return {
      msg,
      changeMsg
    }
  }
}).mount('#app')
```

### 最重要的四個差別

| 比較項目 | 選項式 API | 組合式 API |
| --- | --- | --- |
| 資料放哪裡 | `data()` | `setup()` 內的 `ref()` |
| 方法放哪裡 | `methods` | `setup()` 內的 function |
| 怎麼給畫面用 | Vue 自己整理 | 手動 `return` |
| `this` 會不會出現 | 常出現 | 通常不用 |

結論很簡單：

- 比賽要快，先把組合式 API 看熟
- 看到選項式 API 不要怕，先找 `data` 和 `methods`

## `ref()` 是什麼

`ref()` 可以把資料變成反應式資料。

反應式資料的意思是：

只要資料改了，畫面就會跟著改。

例如：

```js
const score = ref(10)
score.value = 20
```

在 JavaScript 裡改值要用 `.value`。

在 HTML 模板裡不用寫 `.value`，直接寫變數名就好：

```html
<p>{{ score }}</p>
```

### `ref` 和 `reactive` 的白話差別

你可以先把它想成兩種收納方式。

`ref` 比較像「單一抽屜」。

你把一個值放進去，再用 `.value` 去拿它。

它很適合放：

- 一個數字
- 一段文字
- 一個開關
- 一個目前選到的值

例如：

```js
const score = ref(100)
const name = ref('Amy')
const isOpen = ref(true)
```

`reactive` 比較像「整個收納盒」。

你把一組相關資料包在一起，之後直接用點記號改欄位。

例如：

```js
const user = reactive({
  name: 'Amy',
  score: 100,
  class: 'A'
})
```

改資料時會像這樣：

```js
user.score = 120
```

### 實際應用時有什麼影響

如果你現在做的是小功能，而且畫面上是幾個獨立值在改，`ref` 會比較直覺。

因為：

- 看起來清楚
- 拆功能時比較容易
- 傳到別的函式時不容易搞混

如果你現在做的是一大包表單資料，像是：

- 姓名
- 班級
- 分數
- 狀態

這種很多欄位本來就屬於同一個物件，`reactive` 會比較像正常資料結構。

### 今天課程為什麼主線先用 `ref`

因為今天要先把核心觀念練熟。

`ref` 的優點是：

- 初學時比較容易看懂哪個值在變
- 配合 `{{ }}`、`v-model`、`@click` 很直接
- 在比賽小題裡常常已經夠用

所以今天主範例以 `ref` 為主。

但你要知道 `reactive` 不比較高級，它只是更適合管理「一整包資料」。

### 一句話記法

- 單一值，先想 `ref`
- 一整包物件，想到 `reactive`

如果你還在猶豫，先用 `ref` 完成功能，通常不會錯。

## 事件 `@click`

按鈕最常配的是 `@click`。

```html
<button @click="addPoint">加 1</button>
```

```js
function addPoint() {
  score.value++
}
```

### 常犯錯誤

- HTML 寫了 `@click="addPoint"`，但 `return` 沒交出 `addPoint`
- `ref` 變數在 JavaScript 裡忘了寫 `.value`
- 函式名稱前後拼錯

---

## `v-model`

`v-model` 是雙向綁定。

意思是：

- 你輸入框改了，資料會跟著改
- 資料改了，輸入框也會跟著改

例如：

```html
<input v-model="name">
<p>你好，{{ name }}</p>
```

```js
const name = ref('小明')
```

這是做表單時超常用的功能。

---

## `v-if` 和 `v-show`

兩個都能控制顯示，但做法不同。

### `v-if`

`v-if` 是真的加入或移除元素。

```html
<p v-if="isOpen">內容打開了</p>
```

### `v-show`

`v-show` 是元素還在，只是切換 CSS 的顯示狀態。

```html
<p v-show="isOpen">內容打開了</p>
```

### 什麼時候用哪個

- 不常切換，用 `v-if`
- 很常切換，用 `v-show`

---

## `v-for`

`v-for` 用來跑清單。

```html
<li v-for="item in list" :key="item.id">
  {{ item.name }}
</li>
```

### 為什麼要 `:key`

因為 Vue 需要知道每一筆資料是誰。

有 `key` 才能更穩定地更新畫面，也對動畫更重要。

## Vue 動畫基礎

Vue 動畫的入門重點不是花俏，而是你要知道：

- 哪個元件會包住要動的內容
- 哪些 class 會自動加上去
- 進場和離場是不同階段

### `transition` 的基本寫法

```html
<transition name="box">
  <div v-if="isShow" class="panel">內容</div>
</transition>
```

```css
.box-enter-active,
.box-leave-active {
  transition: all 0.4s ease;
}

.box-enter-from,
.box-leave-to {
  opacity: 0;
  transform: translateY(20px);
}
```

### 讀法很簡單

- `name="box"` 代表 class 前綴叫 `box`
- `enter` 是進場
- `leave` 是離場
- `active` 是過程中
- `from` 是剛開始
- `to` 是結束前

### 列表動畫

列表動畫用 `transition-group`。

重點是每一筆都要有穩定的 `key`。

## Ajax 與 fetch

### 什麼是 Ajax

簡單講，Ajax 就是：

網頁不用整頁重整，也能去拿資料。

今天我們不用舊的 `XMLHttpRequest`，直接用 `fetch`。

### 什麼是 JSON

JSON 是一種資料格式。

它很常長這樣：

```json
[
  { "id": 1, "name": "Amy", "score": 95 },
  { "id": 2, "name": "Ben", "score": 88 }
]
```

前端常常拿到 JSON，再把它變成畫面。

---

## 基礎 fetch 流程

今天我們固定用這個流程：

```js
fetch('./data/data.json')
  .then(function(response) {
    return response.json()
  })
  .then(function(data) {
    console.log(data)
  })
  .catch(function(error) {
    console.log(error)
  })
```

### 每一步在做什麼

1. `fetch(url)`：去拿資料
2. 第一個 `then`：把回應轉成 JSON
3. 第二個 `then`：真的拿到資料內容
4. `catch`：失敗時接住錯誤

### 為什麼這裡不教 `async/await`

因為今天的主題已經很多。

先把 `.then()` 的流程讀熟，之後再看 `async/await` 會更穩。

## 常見錯誤整理

### Vue 常見錯誤

- 忘記引入 `vue.3.5.13.js`
- `mount('#app')` 的選擇器寫錯
- `return` 忘記交出資料或函式
- `ref` 在 JavaScript 裡忘記 `.value`
- `v-for` 忘記寫 `:key`

### fetch 常見錯誤

- 用雙擊方式直接開 HTML 導致抓不到資料
- 路徑寫錯
- 忘記 `return response.json()`
- 把第一個 `then` 和第二個 `then` 的工作混在一起
- 錯誤訊息只印在 console，畫面沒有提示

### 動畫常見錯誤

- `transition name` 和 CSS class 名字對不起來
- 只寫 `enter` 沒寫 `leave`
- 想做列表動畫卻忘記 `key`

## 建議教學說法

### 教 Vue 啟動時

可以這樣說：

「先不要管框架很大這件事，今天只看一個畫面怎麼被 Vue 接手。Vue 接手後，我們只改資料，畫面就自己更新。」

### 教組合式 API 時

可以這樣說：

「你把資料和函式都先放進 `setup()`，最後再用 `return` 交給畫面。畫面只拿得到你有交出去的東西。」

### 教 fetch 時

可以這樣說：

「`fetch` 不是一下就拿到資料，它會先拿到回應，再把回應轉成 JSON，最後你才真的拿到可用資料。」

### 教動畫時

可以這樣說：

「Vue 幫你在不同時間點加 class，我們只要把 class 對應的 CSS 寫好，畫面就會動。」
